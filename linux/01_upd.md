## Работа с файлами, директориями и ФС
1. В домашнем каталоге своего пользователя создать директорию `devops-tasks`; в ней
- директории
    - `projects`
    - `mans`
    - `temp`
- файлы
    - `plan.md`
    - `cheets.md`
2. В директории `projects` из первой задачи создать следующу структуру (с проверкой, не существует ли уже `bash-app`)
```
└── bash-app
    ├── logs/
    ├── src/
    └── tests/
```
3. Собрать из двух предыдущих скриптов один, реализующий такую логику
- при запуске скрипта пользователь указывает название проекта
- скрипт проверяет, есть ли уже такой проект в `devops-tasks`:
    - если есть - сообщает об этом пользователю и завершается;
    - если нет - создает структуру каталогов проекта как в задаче 2, но вместо `bash-app` - название проекта, которое ввел пользователь при старте скрипта
- в созданном каталоге проекта создает файл `readme.md` и записывает в него строку `# <имя проекта>`, где `<имя проекта>` - название проекта, которое ввел пользователь при старте скрипта

4. Реализовать задачу резервного копирования проекта с такой логикой
- при запуске скрипта пользователь указывает название проекта
- скрипт проверяет, есть ли уже такой проект в `devops-tasks`:
    - если есть - сообщает об этом пользователю и создает в директории `/opt/backups` архив проекта (для создания архива используем `tar`) с именем, сгенерированным по шаблону `projectName-timestamp.tar`;
    - если нет - сообщает об этом пользователю и завершается

5. Реализовать задачу удаления проекта: логику продумать самостоятельно
6. Изменить логику скрпитов из задач 3-5 таким образом, чтобы они не требовали ввода данных при запуске, а сами запрашивали данные у пользователя

## Работа с пользователями и правами доступа
1. Реализовать задачу проверки того, существует ли пользователь
- при запуске скрипта пользователь указывает имя другого пользователя
- скрипт проверяет, существует ли такой пользователь и сообщает запускавшему пользователю о результате проверки

2. Дополнить предыдущую задачу такой логикой
- если пользователя нет - скрипт создает его и выводит информацию о нем
- если пользователь есть - скрипт сообщает об этом пользователю и завершается

3. Реализовать задачу добавления пользователя в группу
-  при запуске скрипта пользователь указывает имя пользователя и группу, в которую хочет добавить его
-  скрипт проверяет, существует ли такой пользователь и существует ли такая группа
    - если нет - сообщает об этом пользователю и завершает работу
    - если да - добавляет пользователя в группу и сообщает информацию о том, в каких теперь группах состоит пользователь

4. Изменить логику скрпитов из задач 1-3 таким образом, чтобы они не требовали ввода данных при запуске, а сами запрашивали данные у пользователя
